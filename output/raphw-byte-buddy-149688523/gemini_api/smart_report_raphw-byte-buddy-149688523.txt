## Bug Report: Checkstyle Javadoc Missing for FieldProxy.java Constructors

### 1. Title
Build Failure: Checkstyle Javadoc Missing for `FieldProxy.java` Constructors

### 2. Steps to Reproduce
1.  Clone the `raphw/byte-buddy` repository.
2.  Checkout the specific commit `14f2faaea3331b0f0ce5cbe2f53d2b7aaf48015a`.
3.  Execute the Maven build command: `mvn install -DskipTests=true -Dmaven.javadoc.skip=true -B -V`

### 3. Triggering Input
The bug is triggered by the absence of Javadoc comments for two constructors within the `net.bytebuddy.implementation.bind.annotation.FieldProxy.java` file, which violates the project's Checkstyle rules.

### 4. Expected Behavior
The Maven build should complete successfully, indicating that all code adheres to the defined Checkstyle rules, including Javadoc requirements for all public and protected members.

The Maven build fails during the `maven-checkstyle-plugin:2.17:check` goal for the `byte-buddy-dep` project. The build log explicitly reports two "Missing a Javadoc comment" errors for constructors in `net.bytebuddy.implementation.bind.annotation.FieldProxy.java`.
