## Bug Report: FindBugs DMI_RANDOM_USED_ONLY_ONCE Warning Causes Build Failure

### 1. Title
FindBugs `DMI_RANDOM_USED_ONLY_ONCE` warning causes build failure in `TypeResolver.java`.

### 2. Steps to Reproduce
1.  Clone the `raphw/byte-buddy` repository.
2.  Checkout the specific commit `d3caa93230830ab3ace48ef089ec1f21e64eb107`.
3.  Execute the Maven build and verification process using the command: `mvn verify -Pintegration`.

### 3. Triggering Input
The `findbugs-maven-plugin` is configured to run during the `verify` phase of the Maven build. The plugin flags the instantiation and single use of a `java.util.Random` object within the `resolve()` method of `net.bytebuddy.dynamic.TypeResolver$Active` as a `DMI_RANDOM_USED_ONLY_ONCE` warning. Since the build is configured to fail on FindBugs warnings/errors, this triggers the build failure.

### 4. Expected Behavior
The Maven build should complete successfully, and the `findbugs-maven-plugin` should not report any critical bugs or warnings that lead to a build failure. Specifically, the `DMI_RANDOM_USED_ONLY_ONCE` warning should either not be reported or be explicitly suppressed if it's deemed a false positive or acceptable behavior.

### 5. Observed Behavior
The Maven build fails during the `verify` phase for the `byte-buddy-dep` module. The `findbugs-maven-plugin:3.0.3:check` goal reports one bug, `DMI_RANDOM_USED_ONLY_ONCE`, which causes the build to terminate with a `BUILD FAILURE`.

### 8. Patches / Suggested Fixes
The provided code diff contains a patch that addresses this issue by suppressing the FindBugs warning:

```diff
--- a/byte-buddy-dep/src/main/java/net/bytebuddy/dynamic/TypeResolver.java
+++ b/byte-buddy-dep/src/main/java/net/bytebuddy/dynamic/TypeResolver.java
@@ -175,6 +175,7 @@ public interface TypeResolver {
         }\n
 \n         @Override\n+        @SuppressFBWarnings(value = "DMI_RANDOM_USED_ONLY_ONCE", justification = "Avoid thread-contention")\n         public TypeResolver.Resolved resolve() {\n             return new Resolved(new Random().nextInt());\n         }\n```
This patch adds the `@SuppressFBWarnings(value = "DMI_RANDOM_USED_ONLY_ONCE", justification = "Avoid thread-contention")` annotation to the `resolve()` method, indicating that the use of `new Random().nextInt()` is intentional and justified to avoid thread contention, thus preventing the FindBugs plugin from failing the build.
