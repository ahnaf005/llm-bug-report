Title
- Maven build fails due to Checkstyle violations in src/main/java/com/alibaba/druid/filter/index/IllegalSQLIndexFilter.java

Summary
- After applying the changes in the patch (which delete IllegalSQLIndexFilter and its tests and add Hive SQL formatting changes/tests), the CI Maven build fails during the Checkstyle phase. The failure is caused by 9 Checkstyle violations in src/main/java/com/alibaba/druid/filter/index/IllegalSQLIndexFilter.java (trailing whitespace, blank-line-after-opening-brace, WhitespaceAround for ':' tokens, and an EmptyStatement). The build stops with "You have 9 Checkstyle violations." and exits with code 1.

Steps to Reproduce
1. Checkout the commit (or apply the provided diff) that includes the changes in the CODE DIFF (removal of IllegalSQLIndexFilter.java and associated test, plus the other modifications).
2. On a machine with Java 17 and Maven, run:
   ./mvnw -Penable-for-jdk17+,gen-code-cov clean package -B
3. Observe the build fail during the maven-checkstyle-plugin:check phase.

Triggering Input / Conditions
- The CI run invoked: ./mvnw -Penable-for-jdk17+,gen-code-cov clean package -B (Java 17).
- The repository content contains src/main/java/com/alibaba/druid/filter/index/IllegalSQLIndexFilter.java which Checkstyle inspects.
- The code has formatting/style issues that violate the project's src/checkstyle/druid-checks.xml ruleset.

Expected Behavior
- Maven Checkstyle plugin should complete with zero violations and the build should proceed to package the modules.
- If the file was intentionally deleted in this patch, the Checkstyle step should not report violations for that file (i.e., the repo state used by CI should match the intended changes).
