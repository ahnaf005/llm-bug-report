**Bug Report**

**1. Title:** Build Failure: `LibTest` fails due to Logback dependency version mismatch (1.3.14 vs 1.3.15)

**2. Steps to Reproduce:**
1.  Checkout the `GoldenGnu/jeveassets` repository at commit `f91cc245896898e0bc0645b5cac1d3baeef6357b`.
2.  Ensure Java 17 (Temurin distribution) is configured.
3.  Execute the Maven build command: `mvn -B package --file pom.xml -P skip-online-tests`

**3. Triggering Input:**
The bug is triggered by a code change in `src/main/java/net/nikr/eve/jeveasset/LibraryManager.java` where the `logback-core` and `logback-classic` dependencies were updated from version `1.3.14` to `1.3.15`.

**4. Expected Behavior:**
The `LibTest` should pass, indicating that the library versions are correctly managed and the build should complete successfully.

**5. Observed Behavior:**
The build fails during the test phase (`maven-surefire-plugin:2.22.2:test`). Specifically, the `net.nikr.eve.jeveasset.lib.LibTest` fails with an `AssertionError`. The error message indicates that the test detected `logback-core-1.3.15.jar` and `logback-classic-1.3.15.jar` as "added libraries" and `logback-core-1.3.14.jar` and `logback-classic-1.3.14.jar` as "removed libraries", which contradicts the test's internal expectations.

**8. Patches / Suggested Fixes:**
The provided code diff is the change that introduced the new Logback dependency versions (1.3.15). The `LibTest` is currently configured to fail when such a change occurs, likely due to hardcoded expectations for version `1.3.14`.

To fix the build, the `net.nikr.eve.jeveasset.lib.LibTest` needs to be updated to:
*   Expect the new `logback-core-1.3.15.jar` and `logback-classic-1.3.15.jar` versions.
*   Or, be refactored to dynamically check for the presence of the latest versions without hardcoding specific version numbers, allowing for easier dependency updates in the future.
