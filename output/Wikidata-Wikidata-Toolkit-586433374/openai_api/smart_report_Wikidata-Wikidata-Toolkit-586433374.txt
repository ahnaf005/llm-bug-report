Expected Behavior

  - The test should compare the two values in the same representation (for example compare label.getText() to copy.findLabel("fr") if findLabel returns a plain string).

Root Cause Analysis
- The test compares two values in different representations/types:
  - The variable label contains an object whose string representation includes language info (e.g., "\"pleutre\" (fr)").
  - copy.findLabel("fr") returns a plain string "pleutre".
- This leads to a value mismatch in assertEquals, producing the observed assertion failure.
- Two ways to resolve: normalize the result types in the test (preferred for tests), or change the implementation to preserve language metadata if that is the intended behavior.

Patches / Suggested Fixes
1) (Already proposed in code diff) Change the test assertion to match the plain string returned by copy.findLabel:
   - Replace:
       assertEquals(label, copy.findLabel("fr"));
     With:
       assertEquals("pleutre", copy.findLabel("fr"));

   This will make the test compare two plain strings (literal "pleutre" vs returned "pleutre") and should pass given the current behavior.

2) (Alternative, more robust) Compare the textual content explicitly using the label object's text, preserving intent and avoiding brittle string literals:
   - Replace:
       assertEquals(label, copy.findLabel("fr"));
     With:
       assertEquals(label.getText(), copy.findLabel("fr"));
   - This keeps the comparison tied to the expected label text and is less fragile if the expected text changes.

3) (If intended behavior is to preserve language metadata) Change the implementation of copy.findLabel(...) to return a MonolingualTextValue (or the same type as label) rather than plain String, or provide an alternative accessor returning the MonolingualTextValue. Update the test accordingly:
   - Either change findLabel signature or add findLabelValue(String lang) -> MonolingualTextValue.
   - Then the test can safely assert equals(label, copy.findLabelValue("fr")).

Recommendation
- Update the test to compare the same representation. The best immediate fix is to use label.getText() in the assertion so the test checks the textual content instead of relying on toString formatting or different types:
    assertEquals(label.getText(), copy.findLabel("fr"));
- Alternatively, if the intended API is that findLabel returns MonolingualTextValue, modify the API/implementation and update callers/tests accordingly. But that is a bigger API change and may break other users; prefer changing the test.
