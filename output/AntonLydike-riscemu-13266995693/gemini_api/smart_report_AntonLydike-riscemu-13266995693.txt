## Bug Report: NameError: 'Optional' not defined during pytest collection

### 1. Title
`NameError: name 'Optional' is not defined` during pytest collection due to missing `typing.Optional` import.

### 2. Steps to Reproduce
1.  Checkout the repository at commit `92b93ebfcc753030479e1305fd138cc9dea405ce` (the state before the provided patch).
2.  Ensure Python 3.10.11 (or similar version) is installed and active.
3.  Install project dependencies:
    ```bash
    pip install -r requirements.txt -r requirements-dev.txt
    ```
4.  Run the pytest suite:
    ```bash
    pytest -W error
    ```

### 3. Triggering Input
The `pytest -W error` command triggers the test collection process. The bug is caused by the Python interpreter attempting to resolve the type hint `Optional` in `riscemu/types/binary_data_memory_section.py` without `Optional` being imported from the `typing` module.

### 4. Expected Behavior
Pytest should successfully collect all tests and proceed to execute them. There should be no `NameError` during the import phase of the modules.

### 5. Observed Behavior
Pytest fails during the test collection phase with a `NameError: name 'Optional' is not defined`. This error occurs when importing modules that depend on `riscemu/types/binary_data_memory_section.py`, specifically at line 19 within the `BinaryDataMemorySection` class definition.

### 6. Relevant Code Snippets
The issue is in `riscemu/types/binary_data_memory_section.py` at line 19, where `Optional` is used as a type hint without being imported.

**riscemu/types/binary_data_memory_section.py (before fix):**
```python
# ... other imports
from . import (
    MemorySection,
    InstructionContext,
    MemoryFlags,
    SectionType,
)

class BinaryDataMemorySection(MemorySection):
    """
    A memory section that holds binary data.
    """

    def __init__(
        self,
        name: str,
        base_address: int,
        size: int,
        context: InstructionContext,
        owner: str = "root",
        flags: Optional[MemoryFlags] = None,  # <-- Error occurs here
        section_type: SectionType = SectionType.DATA,
    ):
        super().__init__(name, base_address, size, context, owner, section_type)
        self.flags = flags if flags is not None else MemoryFlags.RW
        self.data = bytearray(size)
```
