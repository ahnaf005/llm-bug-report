- Importing the riscemu package triggers module-level imports that load riscemu/types/binary_data_memory_section.py.

8) Patches / Suggested Fixes
- The minimal fix is to import Optional from typing in riscemu/types/binary_data_memory_section.py.

  Example patch (already present in the code diff):
  --- a/riscemu/types/binary_data_memory_section.py
  +++ b/riscemu/types/binary_data_memory_section.py
  @@ -1,3 +1,4 @@
  +from typing import Optional
   from . import (
       MemorySection,
       InstructionContext,
       ...

- Alternative fixes:
  - Use from __future__ import annotations (defer evaluation of annotations) at module top (Python 3.7+). That avoids needing typing names at runtime, but is a broader change that affects all annotations and may hide similar mistakes elsewhere.
  - Fully qualify the annotation using typing.Optional (e.g., flags: "Optional[MemoryFlags]" using string annotations), or import typing as a module (import typing) and use typing.Optional.

- Recommended action:
  - Apply the shown import (from typing import Optional) at the top of riscemu/types/binary_data_memory_section.py (as in the diff).
  - Run pytest locally to confirm the import-time error is fixed.
  - Optionally scan other modules for missing typing imports or consider enabling PEP 563 future annotations across the package if appropriate.

9) Notes / Rationale
- This is a straightforward NameError caused by a missing import for a typing helper used in a class-level annotation. The patch included in the diff already adds the missing import and should resolve test collection errors.
- Double-check other modules for similar missing typing imports; import-time evaluation of annotations can cause failures during test collection if annotations reference undefined names.
