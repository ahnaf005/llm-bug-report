**Detailed Bug Report**

---

**1. Title:** Maven Build Failure: `owlapi-parsers` module fails to compile due to missing `OntologyAxiomPair` symbol after class relocation.

**2. Steps to Reproduce:**
1.  Clone the `owlcs/owlapi` repository:
    `git clone --depth=50 --branch=master https://github.com/owlcs/owlapi.git owlcs/owlapi`
2.  Navigate into the cloned directory:
    `cd owlcs/owlapi`
3.  Checkout the specific commit SHA:
    `git checkout -qf 48d51cbdcce995596a61b0abb745d58cc37cfed4`
4.  Attempt to build the project using Maven:
    `mvn install -DskipTests=true -Dmaven.javadoc.skip=true -B -V`

**3. Triggering Input:**
The bug is triggered by building the project at commit `48d51cbdcce995596a61b0abb745d58cc37cfed4`. This commit includes a change where the `OntologyAxiomPair` class was moved from the `parsers` module's test sources to the `tools` module's main sources, but the `parsers` module's `pom.xml` was not updated to reflect this new dependency.

**4. Expected Behavior:**
The Maven build should complete successfully, compiling all modules and installing the artifacts into the local Maven repository.

**5. Observed Behavior:**
The Maven build fails during the `compile` phase of the `owlapi-parsers` module. The Java compiler reports numerous "cannot find symbol" errors for the `OntologyAxiomPair` class within several files in the `parsers` module. This indicates that the `parsers` module cannot locate the `OntologyAxiomPair` class, which is now part of the `owlapi-tools` module. The build retries twice but fails consistently.

**8. Patches / Suggested Fixes:**
The `OntologyAxiomPair` class was moved from the `parsers` module to the `tools` module. For the `owlapi-parsers` module to compile successfully, it needs to declare a dependency on `owlapi-tools` in its `pom.xml`.

**Suggested Fix:**
Add the following dependency to the `dependencies` section of `parsers/pom.xml`:

```xml
<dependency>
    <groupId>${project.groupId}</groupId>
    <artifactId>owlapi-tools</artifactId>
    <version>${project.version}</version>
</dependency>
```
