Summary
- CI build failed with one test failure: SwaggerSchemaServiceTest.modelConvertersRefsAreResolved.
- Root cause: nested (inner) Java class names are being normalized to use '.' instead of '$' in schema naming, but the test (and code elsewhere) expected the raw class name with '$'. The diff includes a fix that normalizes nested class names to use '.' and updates the test accordingly.

Patches / Suggested Fixes (included in diff)
- The submitted patch already includes a fix:
  - Normalize nested class names in SimpleClassModelConverter when calling schema.name(...) by replacing '$' with '.'.
  - Update the unit test to expect the normalized dotted form.

Recommended alternative improvements
- Use Class.getCanonicalName() or Class.getTypeName() instead of manual replace:
  - canonicalName = rawClass.getCanonicalName(); // returns dotted names for nested classes (may be null for anonymous classes)
  - typeName = rawClass.getTypeName(); // returns a human-readable form
- Rationale: rawClass.getName() returns internal name with '$' for nested classes; getCanonicalName() returns dotted name (or null for anonymous/local). Using getCanonicalName() or getTypeName() would be clearer and less error-prone than manual replace, but be aware canonicalName can be null for certain classes (anonymous local classes).
- Example alternative:
  - String name = rawClass.getCanonicalName();
  - if (name == null) { name = rawClass.getName().replace("$", "."); }
  - schema.name(name);

Action Taken / Patch in PR
- The PR changes both:
  1. Production code: replaces "$" with "." when setting schema name to produce dotted fully-qualified names for nested classes.
  2. Test: updated to assert the dotted name (replace("$", ".") on getName()).
