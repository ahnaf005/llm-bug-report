**Bug Report**

---

**1. Title:** Test Failure: `SwaggerSchemaServiceTest.modelConvertersRefsAreResolved()` expects '$' in inner class name but receives '.'

**2. Steps to Reproduce:**
1.  Execute the Gradle `build` task for the `springwolf-core` project.
2.  Observe the test results, specifically for the `SwaggerSchemaServiceTest` class.

**3. Triggering Input:**
The test method `io.github.springwolf.core.asyncapi.schemas.SwaggerSchemaServiceTest.modelConvertersRefsAreResolved()` is executed. This test involves resolving a schema for an inner class named `ModelConverterNativeClass`.

**4. Expected Behavior:**
The `SwaggerSchemaService` should generate schema names for inner classes using the `$` symbol as a separator, consistent with `Class.getName()`. The test expects the `referencedSchemas` map to contain a key like `io.github.springwolf.core.asyncapi.schemas.SwaggerSchemaServiceTest$ModelConverterNativeClass`.

**5. Observed Behavior:**
The `SwaggerSchemaServiceTest.modelConvertersRefsAreResolved()` test fails with an `AssertionError`. The `actual` schema map generated by the `SwaggerSchemaService` contains a key for the inner class `ModelConverterNativeClass` where the `$` symbol has been replaced by a `.` (e.g., `io.github.springwolf.core.asyncapi.schemas.SwaggerSchemaServiceTest.ModelConverterNativeClass`). This mismatch between the expected key (with `$`) and the actual key (with `.`) causes the assertion to fail.

**6. Relevant Code Snippets:**
The core issue stems from a change in how schema names are generated for native types, specifically inner classes. The `SimpleClassModelConverter` (and an inline converter within the test) was modified to replace `$` with `.` in the class name when setting the schema name. The test's assertion, however, was not updated to reflect this change in the `failed_sha`.

From `springwolf-add-ons/springwolf-common-model-converters/src/main/java/io/github/springwolf/addons/common_model_converters/converters/SimpleClassModelConverter.java` (causing the actual output to use `.`):
```java
// Old code (before fix):
// schema.name(rawClass.getName());

// New code (causing the observed behavior in the failed build):
schema.name(rawClass.getName().replace("$", "."));
```

From `springwolf-core/src/test/java/io/github/springwolf/core/asyncapi/schemas/SwaggerSchemaServiceTest.java` (the failing assertion at line 120 in the `failed_sha`):
```java
assertThat(schema.referencedSchemas()).hasSize(1).containsKey(ModelConverterNativeClass.class.getName());
```
Here, `ModelConverterNativeClass.class.getName()` would return a string containing `$`, which no longer matches the schema name generated by the converter.

**8. Patches / Suggested Fixes:**
The provided code diff contains the necessary patches to align the test's expectations with the schema generation logic. This involves updating both the `SimpleClassModelConverter` and the `SwaggerSchemaServiceTest` to consistently use `.` instead of `$` for inner class names in schema references.
