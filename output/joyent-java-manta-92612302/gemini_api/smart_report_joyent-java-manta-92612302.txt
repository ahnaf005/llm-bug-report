## Bug Report: Compilation Failure - Missing Apache Commons Collections4 Dependency

### 1. Title
Compilation Failure: `org.apache.commons.collections4` package not found in `MantaHttpHeadersIT.java`

### 2. Steps to Reproduce
1.  Clone the `joyent/java-manta` repository.
2.  Checkout the specific commit `be9c5219de8f24d82bba7dcae0fd65598e268aa6`.
3.  Ensure Java 8 (Oracle JDK 1.8.0_31) and Apache Maven 3.2.5 are installed.
4.  Navigate to the project root directory (`joyent/java-manta`).
5.  Execute the Maven install command: `mvn install -DskipTests=true -Dmaven.javadoc.skip=true -B -V`

### 3. Triggering Input
The Maven command:
`mvn install -DskipTests=true -Dmaven.javadoc.skip=true -B -V`

### 4. Expected Behavior
The project should compile successfully, and the Maven build process should complete without errors, indicating all modules are built and installed into the local Maven repository.

### 5. Observed Behavior
The build fails during the `testCompile` phase of the `java-manta-it` module. The Java compiler reports that the package `org.apache.commons.collections4` does not exist and consequently, the `CollectionUtils` symbol cannot be found within `MantaHttpHeadersIT.java`.

An initial attempt also showed a `PluginResolutionException` for `commons-beanutils` due to a connection timeout to `repo.maven.apache.org`, but subsequent retries consistently failed with the compilation error.

### 8. Patches / Suggested Fixes
The provided code diff contains the necessary patches to resolve this issue:
1.  **`pom.xml`**: Add the `dependency.commons-collections.version` property (set to `4.0`) to the parent `pom.xml` to centralize its definition.
2.  **`java-manta-client/pom.xml`**: Remove the `dependency.commons-collections.version` property from this module's `pom.xml` as it is now defined in the parent.
3.  **`java-manta-it/pom.xml`**: Add `org.apache.commons:commons-collections4` as a `test` scoped dependency, using the version defined in the parent POM.

These changes ensure that the `java-manta-it` module has the required `commons-collections4` library available during the test compilation phase, resolving the "package does not exist" and "cannot find symbol" errors.
